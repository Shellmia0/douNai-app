## 1. 项目概述

### 1.1 一句话定义

**DouNai 不是又一个 to-do list，而是一个"探索动力引擎"。**

核心循环：📋 拆任务 → ✅ 完成确认 → 🌸 积攒探索点 → 🗺️ 解锁南京探索

### 1.2 项目背景

- **用户**: 在读硕士研究生，面临论文写作拖延困境
- **痛点**: 现有工具（to-do list、番茄钟、多邻国）无法有效驱动长期行为改变
- **目标**: 打造一个有温度、可长期使用的个人动力工具
- **定位**: 仅服务个人，非商业产品

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| 有温度 | 不是冷冰冰的数据仪表盘，而是有人情味的伙伴 |
| 连接真实世界 | 奖励是真实的探索体验，不是虚拟货币 |
| 轻触式交互 | 嵌入日常但不侵犯边界，不做推送轰炸 |
| 低门槛启动 | 小任务即可获得奖励，避免"大目标瘫痪" |
| 鼓励深度思考 | 不只记录行为，还引导反思和成长 |

## 2. 用户调研

### 2.1 调研方法

通过 Telegram 群组进行了 **14 题结构化问卷**，分三轮完成：

- **第一轮 (Q1-Q6)**: 拖延模式、触发因素、情绪状态、过往干预尝试
- **第二轮 (Q7-Q11)**: 社交驱动力、奖励敏感度、监督偏好、技术平台
- **第三轮 (Q12-Q14)**: 截止日期、当前进度、每日可用时间

### 2.2 关键发现

#### Q7 — 社交驱动力
> "有人每天温和地问一句就是帮助"

**结论**: 用户不抗拒监督，需要的是"有人在看"的回响感

#### Q8 — 奖励敏感度
> "报复性刷手机和玩游戏，想到这些东西反而没有兴奋感"
> "内心特别渴望出去游玩看看自然风光，但总是缺少出去的理由"
> "办的南京公园年卡都快过期了"

**结论**: 传统虚拟奖励（游戏、积分）已燃尽，**真实世界探索**是核心驱动力

#### Q9-Q10 — 监督风格
> "温柔+严厉地循循善诱或许有用"

**结论**: 需要教练型角色，不是朋友也不是老板

#### Q11 — 技术偏好
> "微信、小红书、抖音（已卸载）"
> "工具如果能嵌到习惯中当然很好，但也希望不要过度侵犯导致厌恶"

**结论**: 嵌入日常但有边界感

#### Q12-Q14 — 项目约束
- 初稿截止: **2026年3月9日** (5天冲刺)
- 完成度: ~50%，Ch3/4/5 需打磨补充
- 每日可用: **4-6 小时**
- 最终提交: 6月

## 7. 日记评估引擎

### 7.1 设计理念

> 评估标准透明化 → 用户为了得高分会不知不觉往深处想 → 深度思考自然发生

### 7.2 评分维度（5星制）

| 维度 | 检测方式 | 分值 |
|------|---------|------|
| ✍️ 字数 | 100字+ = 1⭐, 300字+ = 2⭐ | 最多 2⭐ |
| 🔍 反思分析 | 含"因为/发现/意识到/原来/明白了" 等 | 1⭐ |
| 🎯 行动计划 | 含"下次/打算/计划/明天/目标" 等 | 1⭐ |
| ❓ 自我提问 | 含问号 "？" | 1⭐ |

**每⭐ = 0.5 🌸，满分 5⭐ = +2.5 🌸**

### 7.3 交互设计

1. **实时预览**: 写的过程中星级实时更新
2. **引导提示**: 底部显示"再写XX字解锁"或"试试用'因为'"
3. **提交评估**: 弹出详细评估报告，逐项告知得分/失分
4. **修改机制**: 提交后可修改，自动退还旧分数再重新评估

### 7.4 检测关键词表

```
反思词: 因为、所以、发现、意识到、原来、明白了、理解了、
       感受到、反思、思考、领悟、认识到、意味着、说明、
       之所以、根本原因、本质上、实际上、深层

行动词: 下次、以后、打算、计划、明天、接下来、要做、准备、
       目标、改进、调整、尝试、第一步、具体来说、行动
```
